# 01-big-o.md

## 시간복잡도 감 잡기

### 자주 쓰는 기준

- O(1): 해시 조회, 변수 접근
- O(log n): 이진 탐색
- O(n): 단일 루프
- O(n log n): 정렬
- O(n²): 이중 루프
- O(2ⁿ): 완전 탐색 (거의 불가)

### 입력 크기 기준

- n ≤ 100,000 → O(n log n) 가능
- n ≤ 1,000,000 → O(n) 권장
- n ≤ 2,000 → O(n²) 가능
- n ≤ 500 → O(n³) 가끔 가능

### 실수 포인트

- 정렬을 숨겨서 쓰고 있는지 체크
- 재귀 깊이 = 호출 횟수
